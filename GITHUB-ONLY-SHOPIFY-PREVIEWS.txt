SOLUTION 100 % GITHUB PAGES / GITHUB REPOSITORY

Shopify blocks external iframe embedding. GitHub Pages cannot remove Shopify's
X-Frame-Options or Content-Security-Policy headers because GitHub Pages only
serves static files.

This version replaces the two blocked Shopify iframes with screenshots that:
- are generated by GitHub Actions;
- update automatically every 6 hours;
- remain clickable and open the real website;
- require no Cloudflare, Vercel, Netlify or external server.

INSTALLATION
1. Upload every file and folder from this package to the root of your repository.
2. In GitHub, open Settings > Actions > General.
3. Under Workflow permissions, select "Read and write permissions" and save.
4. Open Actions > Update Shopify previews > Run workflow.
5. Wait for the workflow to finish and commit the two WebP images.
6. GitHub Pages will publish the updated previews automatically.

The previews are not interactive inside Studio Rage. A genuinely interactive
Shopify iframe is technically impossible with GitHub Pages alone when the
Shopify store blocks framing.
